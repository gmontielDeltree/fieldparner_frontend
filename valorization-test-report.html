Annual Plan Valorization - Issues Test Report

Date:

URL: http://localhost:5173/init/overview/annual-plan-valorization

Executive Summary

Three critical input issues were identified and fixed in the Annual Plan Valorization page:

  1. Lote (Lot) Select: The select input was clearing immediately after selection
  2. Rinde Histórico Input: Only allowed typing a single digit
  3. Cotiz Futura Cereal Input: Had similar typing restrictions

Issue 1: Lote Select Clearing

Problem Description

When selecting a lot from the dropdown, the value would clear immediately after selection, making it impossible to select a lot.

// ORIGINAL CODE (Problematic) case 'loteId': const selectedField = fields.find(f => f._id === updatedFormData.campoId); const selectedLot = selectedField?.lotes.find(l => l.properties.nombre === value); if (selectedLot) { const newHectareas = selectedLot.properties.hectareas || 0; updatedFormData.has = newHectareas; // MISSING: updatedFormData.loteId = value; // ... } break;

✅ Fix Applied

The issue was that the loteId value wasn't being preserved in the updatedFormData object.

// FIXED CODE case 'loteId': const selectedField = fields.find(f => f._id === updatedFormData.campoId); const selectedLot = selectedField?.lotes.find(l => l.properties.nombre === value); if (selectedLot) { const newHectareas = selectedLot.properties.hectareas || 0; // Important: preserve the loteId value updatedFormData.loteId = value; // ← ADDED THIS LINE updatedFormData.has = newHectareas; // ... } else { // Even if lot is not found, preserve the value to avoid clearing the select updatedFormData.loteId = value; // ← ADDED THIS FALLBACK } break;

Issue 2: Rinde Histórico - Single Digit Input

Problem Description

The input field only allowed typing a single digit. When trying to type "45", only "4" would remain.

// ORIGINAL CODE (Problematic) <TextField type="number" label={t("historical_yield_qq_ha")} value={formData.rindeHistorico} onChange={(e) => handleFieldChange('rindeHistorico', parseFloat(e.target.value) || 0)} />

✅ Fix Applied

The parseFloat was being called on every keystroke, interfering with multi-digit input.

// FIXED CODE <TextField type="number" label={t("historical_yield_qq_ha")} value={formData.rindeHistorico} onChange={(e) => { const value = e.target.value; // Allow typing, only parse when needed handleFieldChange('rindeHistorico', value === '' ? 0 : parseFloat(value) || 0); }} onBlur={(e) => { // Ensure clean number on blur const value = parseFloat(e.target.value) || 0; handleFieldChange('rindeHistorico', value); }} />

Issue 3: Cotiz Futura Cereal Input

Problem Description

Similar to the Rinde Histórico field, this input also had restrictions on typing multiple digits.

✅ Fix Applied

Applied the same solution as Rinde Histórico - parse on blur instead of every keystroke.
